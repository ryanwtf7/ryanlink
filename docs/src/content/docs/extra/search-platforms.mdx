Ryanlink supports a vast array of search platforms. When performing a search using `manager.search()` or `player.search()`, pass a URL or a query string prefixed with one of the following identifiers.

## Standard Platforms

| Platform | Prefix(es) | Notes |
| :--- | :--- | :--- |
| **YouTube** | `ytsearch:`, `yt:`, `youtube:` | Standard video search |
| **YouTube Music** | `ytmsearch:`, `ytm:`, `youtubemusic:` | Music-only search (recommended) |
| **SoundCloud** | `scsearch:`, `sc:`, `soundcloud:` | Track search |
| **Bandcamp** | `bcsearch:`, `bc:`, `bandcamp:` | Track search |

## TTS & Special Sources

| Source | Prefix(es) | Plugin Required |
| :--- | :--- | :--- |
| **Flowery TTS** | `ftts:`, `flowery:` | Built-in |
| **Google Cloud TTS** | `tts:` | `tts-plugin` (DuncteBot) |
| **DuncteBot speak** | `speak:` | `skybot-lavalink-plugin` |
| **PornHub** | `phsearch:`, `pornhub:` | `skybot-lavalink-plugin` |
| **Local File** | `local:` | Node filesystem access |
| **HTTP(S) URL** | `http:`, `https:`, `link:`, `uri:` | Direct URL playback |

## Setting a Default Platform

```typescript
const manager = new RyanlinkManager({
  playerOptions: {
    defaultSearchPlatform: 'spsearch', // Spotify as default
  }
});
```

Any query that isn't a direct URL will automatically use this prefix.

## Source Validation

When `autoChecks.sourcesValidations` is `true` (default), Ryanlink validates that the node has the required source manager enabled before sending the request. Set it to `false` to skip validation:

```typescript
nodes: [{
  id: 'main',
  host: 'localhost',
  port: 2333,
  authorization: 'youshallnotpass',
  autoChecks: {
    sourcesValidations: false,
    pluginValidations: false,
  }
}]
```
