**Ryanlink** includes a built-in, intelligent Autoplay system that automatically finds and adds similar tracks to the queue when current playback ends. This system is designed to be zero-config but highly customizable.

## Quick Start

Enable autoplay globally in your `RyanlinkManager` configuration or per-player.

```typescript
const manager = new RyanlinkManager({
  // ... other options ...
  playerOptions: {
    autoplay: true, // Enable autoplay by default for all players
  }
});
```

---

## Technical Configuration <Badge text="Built-in" variant="tip" />

You can fine-tune how the engine selects new tracks using the `autoplayConfig` object.

| Option | Type | Description |
| :--- | :--- | :--- |
| `enabled` | `boolean` | Whether the system is active. |
| `limit` | `number` | Max number of similar tracks to fetch (Default: 5). |
| `minDuration` | `number` | Minimum track duration (Default: 20,000ms). |
| `maxDuration` | `number` | Maximum track duration (Default: 900,000ms). |
| `defaultSource` | `string` | The search platform to use for discovery (ytmsearch, spsearch). |

### Example Optimization

```typescript
manager.createPlayer({
  guildId: '...',
  autoplay: true,
  autoplayConfig: {
    limit: 10,
    minDuration: 30000,
    excludeKeywords: ['remix', 'slowed', '8d'],
    defaultSource: 'ytmsearch'
  }
});
```

## Intelligence Layers

Ryanlink uses a multi-layered discovery process to ensure high-quality recommendations:

<CardGrid staggered>
  <Card title="Spotify Recommendations" icon="star">
    If the last track was from Spotify, the engine uses the `sprec:` source with the track's ISRC or identifier as a seed.
  </Card>
  <Card title="YouTube Music Similar" icon="random">
    For YouTube tracks, the engine leverages the `RD` (Radio) algorithm to find stylistically identical music.
  </Card>
  <Card title="Artist Discovery" icon="rocket">
    As a fallback, the engine searches the artist's discography on your `defaultSource` to keep the music flowing.
  </Card>
  <Card title="Duplicate Prevention" icon="shieldCheck">
    The engine maintains a high-resolution history to ensure that recently played tracks are never automatically re-added.
  </Card>
</CardGrid>

---

> [!TIP]
> You can toggle autoplay during runtime using `player.setAutoplay(true|false)`. To immediately trigger discovery, use `player.autoplay()`.
