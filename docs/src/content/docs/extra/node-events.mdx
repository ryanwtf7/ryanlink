The `RyanlinkNode` emits events that allow you to monitor the health and connectivity of your audio infrastructure. You can listen to these events on the global `manager.nodeManager` instance.

## Global Node Events

| Event | Payload | Description |
| :--- | :--- | :--- |
| `connect` | `(node: Node)` | Triggered when a WebSocket connection is established. |
| `disconnect` | `(node: Node, reason: string)` | Triggered when a connection is severed. |
| `reconnecting` | `(node: Node)` | Triggered when the manager initiates a reconnection attempt. |
| `error` | `(node: Node, err: Error)` | Triggered when a node encounters a critical REST or WS error. |
| `raw` | `(node: Node, payload: any)` | Triggered for every raw packet received from the node. |

---

## Smart Engine Lifecycle <Badge text="New in 2.0" variant="tip" />

These events are specifically triggered by the Smart Layer to ensure high-availability and state recovery.

### `resumed`
Triggered when a node successfully recovers its session.

```typescript
manager.nodeManager.on('resumed', (node, payload, fetchedPlayers) => {
  console.log(`Node ${node.id} successfully resumed with ${fetchedPlayers.length} players.`);
});
```

### `nodeFailover`
Triggered when an Atomic Migration starts because a node went offline.

```typescript
manager.on('nodeFailover', (player, oldNode, newNode) => {
  console.log(`Successfully migrated player ${player.guildId} from ${oldNode.id} to ${newNode.id}.`);
});
```

---

## Technical Example

```typescript
import { RyanlinkManager, Node } from 'ryanlink';

manager.nodeManager.on('connect', (node: Node) => {
    console.log(`[NODE] ${node.id} is online and ready for playback.`);
});

manager.nodeManager.on('error', (node: Node, error: Error) => {
    console.error(`[NODE] ${node.id} reported an error: ${error.message}`);
});
```

---

> [!TIP]
> Use the `raw` event with caution, as it can be extremely verbose in high-concurrency environments.
