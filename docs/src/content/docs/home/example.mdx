This guide provides a comprehensive example of setting up **Ryanlink** with a modern Discord bot (Discord.js). It showcases the proactive Smart Engine features like zero-config persistence and lifecycle hooks.

## 🚀 Basic Setup (Smart Engine)

The following example initializes the manager with **persistence** and **high-availability** enabled by default.

```typescript
import { RyanlinkManager } from 'ryanlink';
import { Client, GatewayIntentBits } from 'discord.js';

const client = new Client({
  intents: [GatewayIntentBits.Guilds, GatewayIntentBits.GuildVoiceStates],
});

client.ryanlink = new RyanlinkManager({
  nodes: [
    { id: 'main-node', host: 'localhost', port: 2333, password: 'youshallnotpass' }
  ],
  client: { id: '...', username: 'Bot' },
  sendToShard: (guildId, payload) => {
    client.guilds.cache.get(guildId)?.shard.send(payload);
  },
  // --- Smart Engine Options ---
  resume: true,               // Enable automated session persistence
  trackResolveRetryLimit: 3,  // Automated retries for flaky sources
  autoMove: true              // Enable Atomic Migration on node failure
});

// Important: Provide voice updates to Ryanlink
client.on('raw', (d) => client.ryanlink.provideVoiceUpdate(d));

client.on('ready', () => {
  client.ryanlink.init(client.user);
  console.log(`[BOOT] Ryanlink is online.`);
});
```

## 👂 Global Event Listeners

Ryanlink provides a rich event lifecycle to help you monitor your players and nodes.

### Player Lifecycle
```typescript
manager.on('trackStart', (player, track) => {
  console.log(`Playing: ${track.info.title}`);
});

manager.on('queueEnd', (player) => {
  console.log('Finished playing all tracks.');
});

manager.on('queueErrorReport', (player, track, error) => {
  console.error(`Failed to resolve: ${track.info.title} after 3 retries.`);
});
```

### Node Lifecycle
```typescript
manager.nodeManager.on('connect', (node) => {
  console.log(`Node #${node.id} is connected.`);
});

manager.on('nodeFailover', (player, oldNode, newNode) => {
  console.log(`Atomic Migration complete for guild ${player.guildId}.`);
});
```

## 📁 Queue Synchronization (Advanced)

If you need to sync your queue with a dashboard or external storage, use the `QueueChangesWatcher`.

```typescript
import { QueueChangesWatcher } from 'ryanlink';

class MyWatcher implements QueueChangesWatcher {
  tracksAdd(guildId, tracks, ...args) {
    console.log(`${tracks.length} tracks added to guild ${guildId}`);
  }
}

const manager = new RyanlinkManager({
  // ... rest of config ...
  queueOptions: {
    queueChangesWatcher: new MyWatcher()
  }
});
```
