## Core

- **RyanlinkManager** — central hub for nodes, players, voice state, and search
- **Multi-node support** — weighted load balancing (`memory`, `cpuLavalink`, `cpuSystem`, `calls`, `playingPlayers`, `players`, `weighted`)
- **Atomic node migration** — seamless player failover when a node disconnects
- **Session resuming** — Lavalink session persistence across reconnects
- **Queue persistence** — pluggable `QueueStore` (in-memory by default, custom Redis/disk supported)
- **Track resolve retries** — configurable retry limit with `autoSkipOnResolveError`
- **Voice state sync** — automatic `VOICE_STATE_UPDATE` / `VOICE_SERVER_UPDATE` handling

## Player

- **Full playback control** — `play`, `pause`, `resume`, `seek`, `skip`, `stop`, `setVolume`, `setRepeatMode`
- **Repeat modes** — `off`, `track`, `queue`
- **Gapless preloading** (NodeLink) — `nextTrack` field for zero-silence transitions
- **High-Performance Audio** — Native fetch, efficient memory management, and parallel track loading.
- **Dynamic Filter Stacking** — Non-destructive filter layering with built-in presets (Pop, Electronic, etc.).
- **Proactive Automation** — Smart auto-pause/resume, empty channel handling, and resolution self-healing.
- **Deep Lyrics Integration** — Native support for timed and text lyrics via `lavalyrics` plugin.
- **Memory-Optimized Registry** — Industry-leading `TrackRegistry` for zero-overhead large queue management.
- **Strictly Type-Safe** — 100% TypeScript with advanced module augmentation support.
- **Volume decrementer** — scale internal volume independently of reported volume
- **Lifecycle hooks** — `onTrackStart`, `onQueueEnd`, `onNodeFailover` per-player

## Queue

- **TrackRegistry** — memory-efficient track reference mapping for large queues
- **Previous tracks buffer** — configurable `maxPreviousTracks` (default 25)
- **Queue operations** — `add`, `remove`, `splice`, `move`, `shuffle`, `clear`
- **Fisher-Yates shuffle** — statistically unbiased randomization
- **Queue changes watcher** — `QueueChangesWatcher` interface for custom side effects
- **Unresolved tracks** — lazy resolution with `useUnresolvedData` support

## Search

- **50+ search prefixes** — YouTube, Spotify, Apple Music, Deezer, Tidal, SoundCloud, JioSaavn, Yandex Music, VK Music, Qobuz, Pandora, Audiomack, Shazam, Last.fm, Amazon Music, Gaana, Instagram, Bilibili, Bandcamp, and more
- **NodeLink exclusive** — Mixcloud, NicoVideo, Songlink, Audius, Anghami, Bluesky, Letras, Eternal Box, Piper TTS, Google TTS
- **LavaSearch** — `audioSearch()` for albums, artists, playlists, text results
- **Source validation** — configurable `autoChecks.sourcesValidations` and `autoChecks.pluginValidations`
- **Link matchers** — automatic source detection for direct URLs

## Autoplay

- **Multi-source fetch chain** — Spotify rec → SoundCloud → YouTube mix → Deezer → artist search → title search → ytsearch fallback
- **Weighted scoring** — duration similarity (40pts) + same artist (30pts) + partial match (15pts) + jitter (10pts)
- **Circular history buffer** — configurable `historyLimit` (default 20), prevents re-queuing recent tracks
- **Pre-fetch** — triggers early when queue drops to `prefetchThreshold` tracks
- **Keyword exclusion** — regex-based blacklist with modifier-aware logic
- **Duration tolerance** — soft window, not hard exclusion

- **Audio Mixer** — overlay TTS/SFX on main track (`addMixerLayer`, `listMixerLayers`, `updateMixerLayerVolume`, `removeMixerLayer`)
- **Fading** — smooth volume transitions (`trackStart`, `trackEnd`, `stop`, `seek`, `ducking`) with `linear`, `exponential`, `logarithmic`, `s-curve`
- **Gapless preloading** — `setNextTrackGapLess`, `removeNextTrackGapLess`
- **Direct streaming** — `getDirectStream`, `loadDirectStream`, `loadDirectStreamPost`
- **Lyrics** — `loadLyrics`, `loadChapters`, `getMeaning`, `timedLyrics.*`
- **YouTube config** — `updateYoutubeConfig`, `getYoutubeConfig`, `getYoutubeOAUTH`, `updateYoutubeOAUTH`
- **Workers** — `getWorkers`, `patchWorker`
- **Track encoding** — `encodeTrack`, `encodeTracks`
- **Extended stats** — `getDetailedStats` with per-source and per-endpoint tracking
