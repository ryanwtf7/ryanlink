## Manager Configuration

```typescript
import { RyanlinkManager } from 'ryanlink';

const manager = new RyanlinkManager({
  // ── Required ──────────────────────────────────────────────────────────────
  nodes: [
    {
      id: 'main',
      host: 'localhost',
      port: 2333,
      authorization: 'youshallnotpass',
      secure: false,
      // Optional node settings
      retryAmount: 10,
      retryDelay: 5000,
      retryTimespan: 60000,
      heartBeatInterval: 30000,
      closeOnError: true,
      enablePingOnStatsCheck: true,
      // Disable pre-flight plugin/source checks (recommended)
      autoChecks: {
        pluginValidations: false,
        sourcesValidations: false,
      },
    }
  ],
  sendToShard: (guildId, payload) => {
    // For sharded bots: calculate correct shard from guildId
    const totalShards = (client.options.shardCount as number) || client.ws.shards.size || 1;
    const shardId = Number((BigInt(guildId) >> 22n) % BigInt(totalShards));
    const shard = client.ws.shards.get(shardId);
    if (shard) shard.send(payload);
    else client.guilds.cache.get(guildId)?.shard?.send(payload); // fallback
  },
  client: {
    id: 'your-bot-id',
    username: 'YourBot',
  },

  // ── Session Resuming ──────────────────────────────────────────────────────
  resuming: {
    enabled: true,
    timeout: 60000,
  },

  // ── Player Defaults ───────────────────────────────────────────────────────
  playerClass: CustomPlayer,   // optional custom player class
  autoSkip: true,              // auto-skip on track end
  autoSkipOnResolveError: true,
  trackResolveRetryLimit: 3,
  emitNewSongsOnly: false,

  playerOptions: {
    defaultSearchPlatform: 'spsearch',
    useUnresolvedData: true,
    clientBasedPositionUpdateInterval: 100,
    volumeDecrementer: 1,
    requesterTransformer: (requester) => ({ name: requester.username, id: requester.id }),

    onDisconnect: {
      destroyPlayer: false,
      autoReconnect: true,
      autoReconnectOnlyWithTracks: true,
    },

    onEmptyQueue: {
      destroyAfterMs: 20000,
      autoPlayFunction: async (player, lastTrack) => {
        // your autoplay logic
      },
    },

    autoplayConfig: {
      enabled: false,          // ryanlink built-in autoplay class toggle
      limit: 1,
      minDuration: 20000,
      maxDuration: 900000,
      durationTolerance: 90000,
      historyLimit: 20,
      prefetchThreshold: 1,
      excludeKeywords: ['nightcore', 'slowed', 'reverb'],
    },

    minAutoPlayMs: 0,
    enforceSponsorBlockRequestForEventEnablement: true, // auto-call setSponsorBlock on player connect/changeNode
  },

  // ── Queue ─────────────────────────────────────────────────────────────────
  queueOptions: {
    maxPreviousTracks: 25,
    // Built-in stores: MemoryQueueStore (default), LocalDiskQueueStore, RedisQueueStore
    queueStore: new LocalDiskQueueStore('./.ryanlink/queues'),
    queueChangesWatcher: null,
    resuming: {
      enabled: true,   // auto-sync queue from store on player init
      timeout: 60000,
    },
  },

  // ── Links ─────────────────────────────────────────────────────────────────
  linksAllowed: true,
  linksWhitelist: [],
  linksBlacklist: [],

  // ── Advanced ──────────────────────────────────────────────────────────────
  advancedOptions: {
    enableDebugEvents: false,
    maxFilterFixDuration: 600_000,
    debugOptions: {
      logCustomSearches: false,
      noAudio: false,
      playerDestroy: {
        dontThrowError: false,
        debugLog: false,
      },
    },
  },
});
```

## Node Configuration

| Option | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `id` | `string` | `host:port` | Unique node identifier |
| `host` | `string` | — | Lavalink host |
| `port` | `number` | — | Lavalink port |
| `authorization` | `string` | — | Lavalink password |
| `secure` | `boolean` | `false` | Use WSS/HTTPS (port must be 443) |
| `sessionId` | `string` | — | Manual session ID for resuming |
| `regions` | `string[]` | `[]` | Voice regions this node serves |
| `retryAmount` | `number` | `5` | Max reconnect attempts |
| `retryDelay` | `number` | `10000` | Ms between reconnect attempts |
| `retryTimespan` | `number` | `-1` | Window (ms) to count retries (-1 = unlimited) |
| `heartBeatInterval` | `number` | `30000` | WS heartbeat interval |
| `closeOnError` | `boolean` | `true` | Close socket on error |
| `enablePingOnStatsCheck` | `boolean` | `true` | Ping node on stats check |
| `autoChecks.pluginValidations` | `boolean` | `true` | Validate plugin names before requests |
| `autoChecks.sourcesValidations` | `boolean` | `true` | Validate source managers before requests |

## Player Options

| Option | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `defaultSearchPlatform` | `string` | `'ytsearch'` | Default search prefix |
| `useUnresolvedData` | `boolean` | `false` | Use unresolved track metadata |
| `clientBasedPositionUpdateInterval` | `number` | `100` | Position update interval (ms) |
| `volumeDecrementer` | `number` | `1` | Scale internal volume |
| `applyVolumeAsFilter` | `boolean` | `false` | Apply volume as a filter instead of player volume |
| `minAutoPlayMs` | `number` | `10000` | Min ms between autoplay triggers |
| `trackResolveRetryLimit` | `number` | `3` | Per-player resolve retry limit |
| `onDisconnect.destroyPlayer` | `boolean` | `true` | Destroy player on voice disconnect |
| `onDisconnect.autoReconnect` | `boolean` | `false` | Auto-reconnect on voice disconnect |
| `onDisconnect.autoReconnectOnlyWithTracks` | `boolean` | `false` | Only reconnect if queue has tracks |
| `onEmptyQueue.destroyAfterMs` | `number` | `undefined` | Destroy player N ms after queue empties |

## Player Creation

```typescript
const player = manager.createPlayer({
  guildId: '...',
  voiceChannelId: '...',
  textChannelId: '...',
  selfDeaf: true,
  selfMute: false,
  volume: 100,
  node: 'main',           // optional: pin to specific node
  vcRegion: 'us-east',    // optional: prefer nodes in this region
  autoplay: false,
  // Lifecycle hooks
  onTrackStart: (p, track) => {},
  onQueueEnd: (p) => {},
  onNodeFailover: (p, from, to) => {},
  trackResolveRetryLimit: 3,
  // NodeLink options
  smartLeave: false,
  autoPause: false,
  autoPauseOnMute: false,
});

await player.connect();
```

## Queue Stores

Ryanlink ships three built-in `QueueStore` implementations:

```typescript
import { MemoryQueueStore, LocalDiskQueueStore, RedisQueueStore } from 'ryanlink';

// In-memory (default) — no persistence
new MemoryQueueStore()

// Local disk — persists across restarts
new LocalDiskQueueStore('./sessions')

// Redis — for sharded/clustered bots
new RedisQueueStore(redisClient, 'mybot:queue:')
```

Or implement your own by satisfying the `QueueStoreManager` interface (`get`, `set`, `delete`, `stringify`, `parse`).

---

> Always implement `sendToShard` correctly for your Discord library. Without it, Ryanlink cannot communicate with Discord's voice servers.

## DAVE (E2EE Voice) Compatibility

Discord's DAVE protocol (end-to-end encrypted voice) sends a `VOICE_STATE_UPDATE` with `channel_id: null` during the handshake, even when the bot is actively joining a channel. Ryanlink handles this automatically:

- If `VOICE_STATE_UPDATE` arrives with `channel_id: null` but contains a valid `session_id` and the player is already in a channel, Ryanlink updates the session ID and ignores the disconnect signal.
- When sending voice data to Lavalink, `player.voiceChannelId` is used as a fallback if `player.voice.channelId` is temporarily null.

No configuration is required — this is handled internally in `provideVoiceUpdate`.

> **Lavalink requirement:** DAVE support requires Lavalink **4.2.0+**. Older versions cannot connect to DAVE endpoints and audio will silently fail.
