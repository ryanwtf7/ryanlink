Ryanlink uses a highly modular configuration system designed for both simple Discord bots and large-scale, multi-node deployments.

## Manager Configuration

The `RyanlinkManager` is the brain of the engine. It handles node management, voice state synchronization, and global persistence.

```typescript
import { RyanlinkManager } from 'ryanlink';

const manager = new RyanlinkManager({
  nodes: [
    {
      id: 'main-node',
      host: 'localhost',
      port: 2333,
      password: 'youshallnotpass',
      secure: false,
    }
  ],
  client: {
    id: 'your-bot-id',
    username: 'RyanlinkBot'
  },
  sendToShard: (guildId, payload) => {
    // Your library's shard-send logic
    client.guilds.cache.get(guildId)?.shard.send(payload);
  },
  // ---  Options ---
  resume: true,               // Enable automated session recovery
  trackResolveRetryLimit: 3,  // Automated retries for flaky sources
  useMemoryRegistry: true,    // Optimized mapping for massive queues
});
```

### Advanced Manager Options

| Option | Type | Description |
| :--- | :--- | :--- |
| `autoSkip` | `boolean` | Automatically skip to the next track on playback end. |
| `autoMove` | `boolean` | Automatically trigger Atomic Migration on node disconnect. |
| `linksAllowed` | `boolean` | Allow direct URL playback (YouTube, Spotify, etc.). |
| `emitNewSongsOnly` | `boolean` | Prevent duplicate 'trackStart' events during loops. |

## Player Configuration

Players are created per-guild and inherit global settings but can be customized with specific lifecycle hooks.

```typescript
const player = manager.createPlayer({
  guildId: interaction.guildId,
  voiceChannelId: interaction.member.voice.channelId,
  textChannelId: interaction.channelId,
  selfDeaf: true,
  // --- Lifecycle Hooks ---
  onTrackStart: (p, track) => console.log(`Playing: ${track.info.title}`),
  onQueueEnd: (p) => console.log('Queue exhausted'),
  onNodeFailover: (p, oldNode, newNode) => console.log(`Migrated to ${newNode.id}`),
});
```

### Player Persistence

Ryanlink supports multiple storage drivers for zero-config persistence.

<Tabs>
  <TabItem label="In-Memory">

    // Default: Best for small bots

  <TabItem label="Local Disk">

## Search Enhancements

Ryanlink supports over 50+ search prefixes. You can set a default platform to allow users to search without specifying a source.

```typescript
playerOptions: {
  defaultSearchPlatform: 'ytmsearch', // YouTube Music as default
  useUnresolvedData: true,            // High-performance metadata handling
}
```

### Supported Platform Prefixes
- `ytsearch:` / `ytmsearch:` (YouTube / Music)
- `scsearch:` (SoundCloud)
- `spsearch:` / `sprec:` (Spotify / Recommendations)
- `amsearch:` / `dzsearch:` (Apple Music / Deezer)
- `flowery-tts` <Badge text="TTS" variant="note" />
